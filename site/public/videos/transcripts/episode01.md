# What Asylum is

Beginner · Episode 1

## 1. Title card: "Asylum — the Agent Development Environment

This is Asylum. Instead of betting your afternoon on one AI coding agent, you run several at once — and keep the best.

## 2. Split view: one lonely terminal vs. a board of run cards.

A normal IDE is built for one person. A simple agent wrapper stops at one terminal. Asylum is a whole environment for agents — an ADE.

## 3. Three cards labeled with agent names, each showing a branch chip.

You write one prompt and hand it to several agents. Each works in complete isolation, in its own copy of the repo.

## 4. Animated diagram: one repo → three worktrees on three branches.

That isolation comes from git worktrees. Two agents never step on each other's files. Asylum creates and tears them down for you.

## 5. Three words on screen: Project · Task · Run.

Three words run the whole app. A project is a repo. A task is a prompt. A run is one agent's attempt at that task.

## 6. The Diff surface with a green PASS badge and a merge button.

When the runs finish, you compare their diffs, run the project's real checks, and merge the winner back — behind a safe preflight.

## 7. Loop graphic: pose → fan out → track → review → merge.

Pose, fan out, track, review, merge. That's the loop. Run a small tournament, pick the winner.

## 8. End card: "Next: Install and First Launch".

Let's get it running. See you in episode one.
